# Coinbot Project Rules

## 결과 보고 규칙

- 백테스트 결과를 사용자에게 말할 때 **거래 횟수(B/S)를 반드시 포함**한다.
- 수익률, MaxDD, 거래횟수는 항상 세트로 보고한다.

## 실험 → 결과 저장 패턴

백테스트나 분석 실험을 실행한 후 반드시 결과를 저장한다.

### 파일 구조
- 백테스트 코드: `backtest/` (`backtest_*.py`)
- 결과 CSV: `data/results/`
- 분석 문서: `docs/analysis/YYYY-MM-DD-<주제>.md`
- 인덱스: `docs/analysis/INDEX.md`

### 분석 문서 포맷
```markdown
# 제목

- 날짜: YYYY-MM-DD
- 백테스트 코드: `backtest/파일명.py`
- 결과 CSV: `data/results/파일명.csv`

## 개요
한 줄 설명

## 파라미터 / 그리드
테스트한 파라미터 범위

## 결과 요약
핵심 수치 테이블

## 핵심 인사이트
번호 매긴 발견사항 (데이터 근거 포함)

## 한계점
알려진 제약

## 후속 방향
다음에 시도할 것
```

### 인덱스 업데이트
`docs/analysis/INDEX.md` 테이블에 한 줄 추가. 형식:
```
| 날짜 | 카테고리 | 제목 | 파일링크 | 핵심 결과 한 줄 |
```

## 제안 규칙

- 새로운 전략/실험을 "할까요?", "해볼까요?", "시작할까요?" 등으로 제안할 때, **반드시 전략 상세를 먼저 설명**한다:
  - 매수/매도 조건 (구체적 수식/임계값)
  - 그리드 서치 파라미터 범위
  - 대상 종목/데이터
  - 기존 전략 대비 변경점
- 전략 설명 없이 "해볼까요?"만 던지지 않는다.
